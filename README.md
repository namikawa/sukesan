# SUKESAN

**SUKESAN（スケジュール管理ツール）** は、Google カレンダーと連携したスケジュール管理ツールです。秘書のようにスケジュール調整を半自動化することを目的としています。

大きく2つの機能があります。

1. **スケジュール調整（ワンタイム URL）** — 管理者が発行する **1 回限り・発行から 24 時間有効** の URL から、依頼者が Google カレンダーの空き時間を確認し、希望の枠を選んで予定を登録できる。トップページ（`/`）は利用案内のみで、調整自体はワンタイム URL から行います。
2. **Outlook 同期（管理者専用）** — Outlook（Microsoft Graph）側にのみ存在する予定を検知し、選択したものを Google カレンダーへ反映する管理ツール。

ワンタイム URL の発行・一覧・無効化は **管理画面（`/admin`・管理者専用）** から、カレンダー連携や各種設定は **設定画面（`/settings`・管理者専用）** から行います。いずれもパスワードで保護されています。動線は **トップ画面 → 管理画面 → 設定画面** です。

## 画面と権限

| URL | 権限 | 内容 |
| --- | --- | --- |
| `GET /` | 公開 | 利用案内ページ（ワンタイム URL の入手方法・期限の説明） |
| `GET /t/:token` | トークン | スケジュール調整ページ（期間・必要時間を指定して空き候補を表示・登録） |
| `POST /schedule` | トークン | 選んだ空き枠を管理者カレンダーへ登録し、使用したトークンを無効化 |
| `GET /admin` | 管理者 | 未ログインならログイン画面、ログイン済みならワンタイム URL の発行・一覧・無効化 |
| `POST /tickets` | 管理者 | ワンタイム URL を発行 |
| `POST /tickets/:token/revoke` | 管理者 | 発行済みワンタイム URL を手動で無効化 |
| `GET /settings` | 管理者 | 設定画面（カレンダー連携・調整可能な時間帯など） |
| `POST /settings` | 管理者 | 調整可能な時間帯・曜日・昼休憩の保存 |
| `GET /sync` ほか同期系 | 管理者 | Outlook → Google の同期ツール |
| `/auth/google`, `/auth/microsoft` | 管理者 | カレンダー連携（OAuth） |

- **公開** = 認証不要。**トークン** = 管理者が発行した有効なワンタイム URL（トークン）が必要だが、管理者ログインは不要。**管理者** = 管理者パスワード（`ADMIN_PASSWORD_DIGEST`）によるログインが必要。
- 未認証で管理者ページにアクセスすると、ログイン画面（`/admin`）へ誘導されます。ログイン・ログアウト後はいずれも `/admin` に戻ります。
- Google のトークンはサーバ側（`data/google_token.json`）で共有保存され、ワンタイム URL の利用者全員が「管理者が連携した1つの Google カレンダー（`primary`）」を参照します。
- Microsoft（Outlook）のトークンは管理者のセッションにのみ保持され、Outlook 同期でのみ使用します。

## ワンタイム URL の仕様

スケジュール調整は、管理者が発行する **ワンタイム URL** からのみ行えます。

- 管理者は管理画面（`/admin`）の「新しい URL を発行」で発行します（Google 連携が未設定だと発行できません）。
- 発行された URL は一覧に表示され、有効なものは **「コピー」ボタン**でクリップボードへコピーできます。
- **有効期限は発行から 24 時間**。期限を過ぎると自動的に無効になります。
- **1 回限り**: 予定が登録されると、その URL は使用済みとなり再利用できません。
- 管理者は発行済み URL の状況を一覧（表形式）で確認できます。ステータスは **有効 / 使用済み / 期限切れ / 無効化** の 4 種で、使用済みの行には登録された **依頼者名・予定名・予約枠・登録日時** を表示します。
- 誤発行などに備え、管理者は有効な URL を手動で **無効化** できます。
- 無効・期限切れ・使用済み・存在しない URL にアクセスすると、その旨を案内するページ（HTTP 410）を表示します。
- 発行済み URL は `data/tickets/` に保存されます（ISO 週ごとのファイルに分割。約 30 日経過した古い週ファイルは自動削除）。

## スケジュール調整の仕様

調整画面はワンタイム URL（`/t/:token`）を開くと表示されます。

- 依頼者は **開始日〜終了日** の期間と **必要な時間（分）** を入力します。
- アプリは管理者カレンダーを確認し、**空き時間の候補**を 30 分刻みで算出して **日付ごとに一覧表示**します。
- 候補から希望の枠を選び、**依頼者名**・**予定名**を入力して「スケジュール登録する」を押すと、管理者カレンダーに予定が登録されます。
  - 登録される予定のタイトル: `[予定名] - [依頼者名] (from 調整ツール)`
  - 予定の説明欄: `依頼者: [依頼者名]`
  - 登録が完了すると、その URL は使用済みになり再度は使えません。
- 検索（空き時間のチェック）だけでは URL は無効になりません。希望する枠がなければ、期間・時間を変えて何度でも再調整できます。
- 初回表示の既定値は **翌営業日**・**30 分**です（自動検索はせず、「空き時間をチェック」を押して検索）。

### 調整可能な時間帯・曜日・昼休憩（設定画面）

- **時間帯（営業時間）**: 開始・終了時刻を指定。この範囲内のみ候補を出します（既定 09:00〜18:00）。
- **曜日（営業日）**: チェックを入れた曜日のみ調整可能（既定は平日 月〜金）。
- **昼休憩**: 休憩を確保する時間帯（開始〜終了）と必要な休憩時間（分）を指定（既定 11:00〜14:00 / 60 分）。詳細は「ランチタイム保護」を参照。
- 日付の既定値「翌営業日」も、この曜日設定に従います。
- 設定は `data/settings.json` に永続化されます（サーバ再起動後も保持）。

### 期間の上限

- 一度に表示するのは **最大 5 営業日** です。範囲がそれより長い場合は、最初の 5 営業日分のみ表示し、その旨を画面に注記します。
- カレンダーの取得は範囲全体で 1 回にまとめて行います。

### ランチタイム保護

- 設定した **休憩時間帯**（既定 11:00〜14:00）のうち、**連続した休憩時間**（既定 60 分）を昼休憩として確保するための配慮です。時間帯・分数は設定画面で変更できます。
- ある候補枠を入れると、休憩時間帯内に必要分の連続空きが残らなくなる場合、その枠に **`（ランチタイム）`** タグを付けて控えめに表示します。
- 候補自体は消さないため、急ぎの場合は選択できます。
- **休憩時間を 0 分にすると無効化**できます。

### スパム対策・アクセス制御

- スケジュール調整は、有効なワンタイム URL（トークン）を持つ依頼者のみが利用できます。誰でも登録できる公開フォームではありません。
- **依頼者名は必須**（サーバ側でも検証）。
- **レート制限**:
  - `POST /schedule`（予定登録）は同一 IP から **60 秒間に 5 回まで**。
  - 空き時間検索（`GET /t/:token` で日付・時間を指定した時のみ。Google API を消費する）は同一 IP から **60 秒間に 10 回まで**。これにより、ワンタイム URL の漏えい先などが登録せずに検索を連打して API クォータを消費・遅延させる簡易 DoS を抑止します。ページを開くだけ（検索条件なし）は消費しません。
  - いずれも超過時は 429 を返します（プロセス内メモリ管理のため、再起動でリセット。複数プロセス構成では共有されません）。
- サーバ側でトークンの有効性と、選択された空き枠（営業時間・曜日・空き状況）を再検証します（クライアント提示値を信用しません）。
- 二重登録を防ぐため、カレンダー登録の前にトークンを使用済みにし、登録に失敗した場合のみ自動で有効へ戻します。

## Outlook 同期の仕様（管理者専用）

> ⚠️ **この機能は作成途中であり、動作の十分な確認が取れていません。** 仕様・挙動は変更される可能性があり、現時点では本番利用を推奨しません。利用は自己責任でお願いします。

- Google・Outlook の両方を連携し、「2つのカレンダーをチェック」で **Outlook 側にのみ存在するイベント** を抽出します（ステータス: 未反映）。
- 一覧から対象にチェックを入れ、「選択したイベントを Google に同期」で Google カレンダー（`primary`）へ登録します（ステータス: 反映済み）。
- 突き合わせは ID ではなく「件名 + 開始 + 終了」で行います（カレンダー間で ID が異なるため）。
- 取得対象期間は「1 日前 〜 60 日後」（`app.rb` の `SYNC_WINDOW_PAST` / `SYNC_WINDOW_FUTURE`）。同期は Outlook → Google の一方向のみです。

## セットアップ

### 1. 依存 gem のインストール

```bash
bundle install
```

### 2. OAuth クライアントの準備

#### Google Cloud Console

1. プロジェクトを作成し、「Google Calendar API」を有効化
2. OAuth 同意画面を構成（テスト ユーザーに自分のアカウントを追加）
3. 認証情報 → OAuth クライアント ID（種類: ウェブ アプリケーション）を作成
4. 承認済みのリダイレクト URI に以下を登録
   - `http://localhost:3000/auth/google/callback`
5. 発行されたクライアント ID / シークレットを控える
6. 必要なスコープ: `https://www.googleapis.com/auth/calendar.events`（読み取り + イベント作成）

#### Azure Portal（Microsoft Entra ID）※ Outlook 同期を使う場合のみ

1. 「アプリの登録」で新規アプリを作成
2. リダイレクト URI（種類: Web）に以下を登録
   - `http://localhost:3000/auth/microsoft/callback`
3. 「証明書とシークレット」でクライアント シークレットを発行（控えるのは「シークレット ID」ではなく **値**）
4. API のアクセス許可で Microsoft Graph の **委任されたアクセス許可** を追加
   - `Calendars.Read`
   - `offline_access`
5. クライアント ID / シークレット / テナント ID を控える

### 3. 環境変数の設定

`.env.example` をコピーして `.env` を作成し、控えた値を設定します。

```bash
cp .env.example .env
```

```
GOOGLE_CLIENT_ID=...
GOOGLE_CLIENT_SECRET=...
MS_CLIENT_ID=...
MS_CLIENT_SECRET=...
MS_TENANT_ID=common
ADMIN_PASSWORD_DIGEST='$2a$12$....'
SESSION_SECRET=...
```

- `ADMIN_PASSWORD_DIGEST` は設定画面・カレンダー連携・Outlook 同期を保護する管理者パスワードの **bcrypt ダイジェスト**です（平文は保存しません）。**未設定（空）の場合はログインできません。**
  - 生成: `bin/admin_password_digest` を実行し、プロンプトにパスワードを入力（エコーされません）。出力された行を `.env` に貼り付けます。
  - 値に `$` を含むため、**必ずシングルクォートで囲んでください**（`dotenv` の変数展開で値が壊れるのを防ぐため）。
- `SESSION_SECRET` はセッション Cookie の署名鍵です。**本番（`APP_ENV=production`）では必須**で、未設定だと起動に失敗します。開発時は未設定でも一時鍵が自動生成されます（再起動で無効化）。生成例: `ruby -rsecurerandom -e 'puts SecureRandom.hex(64)'`
- `APP_TIMEZONE` は空き時間の算出・表示・入力解釈に使うタイムゾーンです（既定 `Asia/Tokyo`、tz database 名で指定）。起動時にプロセスの `TZ` へ適用し、サーバ OS の設定に依存させません（特定地域での運用を前提）。設定したタイムゾーンは調整画面・設定画面に注記として表示されます。
- `.env` 自体のファイル権限は `chmod 600 .env` で本人のみ読めるようにしておくことを推奨します。

### 4. 起動

```bash
bundle exec rackup -p 3000
# もしくは
ruby app.rb
```

ブラウザで <http://localhost:3000> を開きます。

開発時（既定 `development`）は HTTP のまま利用できます。本番として動かす場合は `APP_ENV=production` を設定してください。本番モードでは次が有効になります。

- **HTTPS 必須**: HTTP アクセスは HTTPS へリダイレクト（TLS は前段のリバースプロキシで終端し、`X-Forwarded-Proto` を渡す構成を想定）。
- **Cookie 強化**: セッション Cookie に `Secure` / `HttpOnly` / `SameSite=Lax` を付与（`SESSION_SECRET` で署名）。
- **エラー秘匿**: スタックトレースを表示せず汎用メッセージを返す。
- **HSTS**: `Strict-Transport-Security` を付与。

### サーバの制御（bin/server）

起動・停止・再起動・状態確認をまとめて行える制御スクリプトを用意しています。

```bash
bin/server start     # 起動（バックグラウンド）
bin/server stop      # 停止
bin/server restart   # 再起動
bin/server status    # 起動状態の確認
bin/server run       # フォアグラウンド実行（サービス管理から起動する用途）
```

- PID は `tmp/pids/server.pid`、ログは `log/server.log` に出力します（いずれも `.gitignore` 済み）。
- ポートは環境変数で変更できます: `PORT=8080 bin/server start`。
- `PID_FILE` / `LOG_FILE` でファイルパスも上書きできます。

### OS サービスへの登録

`deploy/` にサービス定義のテンプレートを同梱しています。`bin/server run`（フォアグラウンド実行）を起動コマンドとして使う想定です。

- **Linux（systemd）**: `deploy/sukesan.service`
- **macOS（launchd）**: `deploy/com.sukesan.server.plist`

いずれもファイル内のコメントに沿って `__APP_DIR__` などのプレースホルダを実環境の値に置き換えてから登録してください。

## 使い方

### 管理者（初回準備〜URL 発行）

1. <http://localhost:3000/admin> を開き、管理者パスワードでログイン
2. 「設定画面へ」から設定画面（`/settings`）を開き、「Google と連携」でカレンダーを接続（以降、ワンタイム URL の利用者全員がこの連携を使用）
3. 設定画面で「調整可能な時間帯・曜日・昼休憩」を設定して保存
4. （任意）Outlook 同期を使う場合は `/sync` で Outlook を連携
5. 管理画面（`/admin`）に戻り「新しい URL を発行」、一覧の「コピー」で URL を取得して依頼者へ共有
6. 一覧で各 URL の状況を確認。誤発行などは「無効化」で停止できる

### 依頼者（スケジュール調整）

1. 管理者から受け取ったワンタイム URL を開く
2. 開始日・終了日・必要な時間を入力し「空き時間をチェック」
3. 日付ごとに表示される候補から希望の枠を選択
4. 依頼者名・予定名を入力し「スケジュール登録する」を押す
5. 管理者カレンダーに予定が登録される（以降、その URL は使用済みになり使えません）

## データと機微情報

- `data/settings.json` — 調整可能な時間帯・曜日・昼休憩の設定。
- `data/google_token.json` — 管理者の Google OAuth トークン（**refresh token を含む機微情報**）。
- `data/tickets/` — 発行済みワンタイム URL の状態と、登録時の入力値（依頼者名・予定名・予約枠）。ISO 週ごとのファイルに分割して保存します。

## 注意・制約

- Google のトークンはファイルで共有保存しますが、Microsoft のトークンと管理者ログイン状態はサーバのメモリ（`Rack::Session::Pool`）に保持します。プロセス再起動で失われます。
- スケジュール調整は、管理者が発行したワンタイム URL を持つ依頼者のみが利用できます（管理者ログインは不要）。トークンは推測困難な乱数ですが、URL を知っている人は期限内・未使用であれば登録できるため、共有相手にご注意ください。
- レート制限はプロセス内メモリのため、複数プロセス構成では共有されません。発行済みワンタイム URL（`data/tickets/`）はファイル保存のため複数プロセスでも共有されます。
- 反映先は Google の主カレンダー（`primary`）です。
- 本番公開時は HTTPS を必須とし、OAuth リダイレクト URI も本番ドメインに合わせて登録してください。
